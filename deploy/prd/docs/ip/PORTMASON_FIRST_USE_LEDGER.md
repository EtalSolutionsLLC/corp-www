## Operating rules

- Use ™ for unregistered software, product, and product-family marks.
- Use ℠ when a mark is used specifically to identify a service.
- Do not use ® unless a federal registration has issued for the applicable goods or services.
- Do not use “Patent Pending” unless an application covering the identified technology has been filed and remains pending.
- Preserve dated specimens, URLs, screenshots, source-control commits, release artifacts, proposals, invoices, and customer-facing documentation.
- A source commit establishes authorship and development chronology; public use and use in commerce must be recorded separately and truthfully.

## Release evidence checklist

For each public Portmason release, record:

1. Release/build number and UTC publication time.
2. Source commit and immutable artifact SHA-256.
3. Public URL and access date.
4. Full-page screenshots showing the mark associated with the relevant product or service.
5. Archived HTML/PDF specimen.
6. Any proposal, agreement, invoice, or customer delivery associating the mark with goods or services.
7. Whether the release constitutes first public use, first use in commerce, or neither.

## Patent review boundary

Potentially patentable subject matter must be documented in a separate confidential invention disclosure before public technical detail is expanded. No Portmason page or artifact may state “Patent Pending” until counsel confirms that a qualifying application has been filed and remains pending.
